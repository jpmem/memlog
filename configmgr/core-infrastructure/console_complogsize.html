コンポーネントに対応するログ サイズの変更手順について

サイト サーバー内では数多くのコンポーネントが動作しており、コンポーネント毎に出力されるログが異なります。
問題の調査のために、特定のコンポーネントに対応するログ サイズの変更が必要な場合、以下の手順でログ サイズを変更してください。
調査対象コンポーネントは、担当者からご案内いたします。

作業端末

Configuration Manager コンソールが動作する端末

実行手順

  1. [Configuration Manager コンソール] - [監視] - [概要] - [システムのステータス] - [コンポーネントのステータス] と遷移します。
  2. 一覧から、サイト システム列が調査対象サーバーとなっているコンポーネントを右クリックして [開始] - [Configuration Manager サービス マネージャー] をクリックします。
  3. 左ペインの [コンポーネント] を展開して、対象コンポーネントをダブルクリックします。
  4. 右ペインの対象コンポーネントを右クリックして [ログの記録] をクリックします。
  5. ログのサイズを変更して [OK] をクリックします。

データベース レプリケーション問題発生時の情報採取手順について

中央管理サイトとプライマリ サイト間で、データベース レプリケーションのリンク状態が「失敗」になるなど、レプリケーションの問題が発生した時の情報採取手順です。

作業端末

  • 中央管理サイト サーバー
  • プライマリ サイト サーバー
  • 中央管理サイトのデータベース サーバー
  • プライマリ サイトのデータベース サーバー

採取情報

  1. spDiagDRS 情報
  2. vLogs 情報
  3. データベース レプリケーションの送信･受信履歴情報
  4. レプリケーション データの初期化ステータス情報
  5. Configuration Manager サーバー情報
